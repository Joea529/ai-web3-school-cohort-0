# 学习计划

> 创建日期：2026-05-18  
> 定位：产品 / 商业分析 / 战略方向  
> 节奏：每天 30 分钟 ~ 1 小时

## 四层学习路径

按 Handbook 的四层结构循序渐进：

### Layer 1：AI 基础（第 1-3 周）

**目标**：建立对 AI 核心概念的系统理解，补齐理论短板。

| 章节 | 优先级 | 说明 |
|------|--------|------|
| 大语言模型（LLM） | ⭐⭐⭐ | 你已经用过，但需要系统理解能力边界 |
| 提示词（Prompt） | ⭐⭐⭐ | 直接影响你每天用 AI 的效率 |
| 上下文（Context） | ⭐⭐⭐ | 理解模型"看见"和"忘记"的机制 |
| 检索增强生成（RAG） | ⭐⭐ | 你实习中用过，加深理解 |
| 智能体（Agent） | ⭐⭐⭐ | 核心概念，你已经在用 Hermes Agent |
| 框架（Frameworks） | ⭐ | 了解即可，不深入开发 |
| 氛围编程（Vibe Coding） | ⭐⭐ | 你已经开始用了 |
| 模型上下文协议（MCP） | ⭐⭐ | 理解 Agent 如何连接工具 |
| 评估 / 微调 / 推理 | ⭐ | 了解概念，不需要实践 |

### Layer 2：Web3 基础（第 4-6 周）

**目标**：从零建立 Web3 基础认知，重点在与 AI 交叉的部分。

| 章节 | 优先级 | 说明 |
|------|--------|------|
| 密码学（Cryptography） | ⭐⭐⭐ | Web3 的根基 |
| 钱包（Wallet） | ⭐⭐⭐ | 理解用户入口，与产品设计直接相关 |
| 智能合约（Smart Contract） | ⭐⭐ | 理解"可编程信任" |
| 开发栈（Dev Stack） | ⭐ | 了解工具链即可 |
| 网络（Network） | ⭐⭐ | 理解链的运作方式 |
| 账户抽象（Account Abstraction） | ⭐⭐ | 提升用户体验的关键技术 |
| DeFi / 预言机 / 索引 / 安全 | ⭐ | 按兴趣选读 |

### Layer 3：AI × Web3 Bridge（第 7-10 周）

**目标**：理解两个领域的交叉点，这是核心价值所在。

| 章节 | 优先级 | 说明 |
|------|--------|------|
| 链感知上下文 | ⭐⭐⭐ | AI 如何"理解"链上世界 |
| Web3 工具调用 | ⭐⭐⭐ | Agent 如何操作链上 |
| 智能体工作流 | ⭐⭐⭐ | 多 Agent 协作模式 |
| 智能体钱包 | ⭐⭐⭐ | 产品设计的核心 |
| 机器支付 | ⭐⭐⭐ | 未来的商业基础设施 |
| 结算与托管 | ⭐⭐ | 商业合规视角 |
| 智能体身份 | ⭐⭐ | 信任体系的基础 |
| 治理 AI | ⭐⭐ | 战略方向重点关注 |
| 去中心化 AI | ⭐⭐ | 行业趋势 |

### Layer 4：前沿探索（持续）

**目标**：保持对前沿的感知，为战略判断积累素材。

| 章节 | 优先级 |
|------|--------|
| 智能体商业 | ⭐⭐⭐ |
| 开发工具 | ⭐ |
| 钱包与权限 | ⭐⭐ |
| AI 安全 | ⭐⭐ |
| 治理 | ⭐⭐ |
| 开放赛道 | ⭐⭐ |
