## 第一性原理

Prompt 是软约束，不是安全边界。真正的边界必须由代码、权限、校验和审计来承担。

## 4 个知识节点

### 1. Instruction
把任务拆成四段：任务目标 → 可用输入 → 禁止行为 → 输出格式和失败格式。区分「解释」和「执行」——研究助手可以整理资料，但不能假装结论已验证。

### 2. Few-shot
用示例教模型处理「边界难描述」的任务。但要当测试用例维护——协议升级后旧示例可能误导模型。

### 3. Structured Output
让模型按固定结构返回（JSON schema、函数参数），后续代码才能检查、拒绝、记录。

### 4. Prompt Injection
攻击者通过用户输入、网页、文档让模型忽略原始规则。应对方案：标记不可信数据 + 参数校验 + allowlist + human approval，不能靠一句「不要被注入」。

## AI × Web3 中的位置

Prompt 处在用户目标和模型行为之间。但安全链路应该是：
Prompt 定义任务 → Context 提供可信数据 → Model 生成候选 → Code 校验 → Guard 检查链上 → Human check
